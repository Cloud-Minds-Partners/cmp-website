## Tasks

### Task 1: Audit current workflow + harden

Read `.github/workflows/deploy.yml` (current). Look for:
- Silent skip patterns (`if: secrets.X` without else error, or `continue-on-error: true`)
- Token reference patterns

Rewrite or patch to:
- Pre-flight step: assert required secrets present (`FIREBASE_SERVICE_ACCOUNT_DCPLATFORMCMP` or whatever name is used). Fail-fast with clear error.
- Use `firebaseHosting-merge` or `firebaseHosting-pull-request` action OR Firebase CLI directly with `--token` if classic.
- For non-main branch: deploy to preview channel with `firebase hosting:channel:deploy <branch> --expires 7d`
- For main branch: deploy to live channel.
- Remove any `continue-on-error: true` on critical steps.

### Task 2: Lighthouse CI step

Add to deploy.yml: after build (and before deploy on main, or as separate job):
```yaml
- name: Lighthouse CI
  run: |
    npm install -g @lhci/cli
    lhci autorun --collect.staticDistDir=./dist --assert.preset=lighthouse:no-pwa --assert.assertions.categories:performance=warn --assert.assertions.categories:accessibility=error --assert.assertions.categories:seo=error
```

Or simpler approach: configure `.lighthouserc.json` with thresholds (perf ≥0.9, a11y ≥0.95, seo ≥0.95) and call `lhci autorun`. If `@lhci/cli` install fails in CI, document workaround: post-deploy manual check.

Acceptable: if @lhci/cli proves brittle in CI, fall back to running `lighthouse` CLI in CI with `--chrome-flags="--headless"` against `npx astro preview &` (background server).

### Task 3: Preview channel per branch

In deploy.yml, branch logic:
```yaml
- if: github.ref != 'refs/heads/main'
  name: Deploy preview channel
  uses: FirebaseExtended/action-hosting-deploy@v0
  with:
    repoToken: '${{ secrets.GITHUB_TOKEN }}'
    firebaseServiceAccount: '${{ secrets.FIREBASE_SERVICE_ACCOUNT_DCPLATFORMCMP }}'
    projectId: dcplatformcmp
    target: cmp-website
    expires: 7d
    channelId: 'preview-${{ github.head_ref || github.ref_name }}'

- if: github.ref == 'refs/heads/main'
  name: Deploy live
  uses: FirebaseExtended/action-hosting-deploy@v0
  with:
    repoToken: '${{ secrets.GITHUB_TOKEN }}'
    firebaseServiceAccount: '${{ secrets.FIREBASE_SERVICE_ACCOUNT_DCPLATFORMCMP }}'
    projectId: dcplatformcmp
    target: cmp-website
    channelId: live
```

Action emits comment with preview URL on PR; verify firebase target `cmp-website` matches `firebase.json` configuration.
