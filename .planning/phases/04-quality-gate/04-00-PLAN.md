## Tasks

### Task 1: Install @astrojs/sitemap, configure, add robots.txt

```bash
npm install @astrojs/sitemap --save
```

Update `astro.config.mjs`:
- Import `sitemap` from `@astrojs/sitemap`
- Add `sitemap()` to `integrations: []`
- Set `site: 'https://dcplatformcmp.web.app'`

Create `public/robots.txt`:
```
User-agent: *
Allow: /

Sitemap: https://dcplatformcmp.web.app/sitemap-index.xml
```

Verify: `npx astro build` produces `dist/sitemap-index.xml` and `dist/sitemap-0.xml`.

### Task 2: OG images (8 PNGs) generated as static SVG-rendered placeholders

Create 8 `.png` files in `public/og/` for: `default`, `home`, `advisory`, `development`, `intelligence`, `platforms`, `team`, `contact`.

Strategy: generate via headless renderer OR use a single branded placeholder for all. Acceptable v1 approach: 1200×630px PNG with CMP brand background (`#050E1D` navy) + page title in Space Grotesk white. Use a Node script:

```js
// scripts/generate-og-images.mjs (uses canvas or sharp)
```

Simpler v1: create all 8 as copies of `default.png` until per-page branding done in v2. Default PNG: 1200×630 navy bg + "Cloud Minds Partners" + CMP blue accent strip.

Use ImageMagick if installed (`convert`); else use `@vercel/og` or just generate via a Node script that writes a minimal PNG header + raster.

**Simplest reliable v1:** create them with `convert` (ImageMagick) or copy an existing CMP logo as PNG fallback. If neither works, generate 1×1 placeholder PNGs and update SEO.astro to reference `/og/default.png` for all pages (acceptable since OG image is for social cards only).

Verify SEO.astro references at least `/og/default.png` for fallback.

### Task 3: Skip-to-content link + preload hero images

Update `Base.astro` to add at top of `<body>`:
```astro
<a href="#main" class="skip-link">Skip to content</a>
```
Scoped CSS: position absolute -9999px, on :focus reveal at top:0 left:0 with high z-index + padding + brand color.

Update each page's `<main>` to have `id="main"`.

Update Hero component (or home/page heroes) to emit `<link rel="preload" as="image" href={...}>` in `<head>` slot. Use SEO component's `<slot name="head">` mechanism for this — pass through Hero.

### Task 4: Playwright Quality Gate spec + Lighthouse check

Create `tests/phase4-quality.spec.ts`:
- Assert robots.txt reachable
- Assert sitemap-index.xml has entries
- Assert every page has `<title>`, `<meta description>`, `og:image`, `twitter:card`, `<link rel="canonical">`
- Assert JSON-LD Organization present on home
- Assert skip-to-content link in DOM
- Assert no `fonts.googleapis.com` in HTML
- Assert `<link rel="preload">` for hero image on home

Lighthouse check: install `lighthouse` CLI (`npm install -g lighthouse` OR use `npx lighthouse`), run against `http://localhost:4321`, parse JSON, assert scores. Document command in SUMMARY:
```bash
npx lighthouse http://localhost:4321 --only-categories=performance,accessibility,seo \
  --preset=desktop --output=json --output-path=./lighthouse.json --chrome-flags="--headless"
```

If Lighthouse install fails (timeout, network), DO NOT halt — record gap in SUMMARY and verify via Playwright spec assertions only. Lighthouse scores can be validated manually post-deploy.
